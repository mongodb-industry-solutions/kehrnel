1. Select the UID of All Vaccination List Compositions

This is the most fundamental query. It retrieves the unique ID (uid) for every composition that matches the specified archetype ID.

SELECT
  c/uid/value
FROM
  EHR e
  CONTAINS COMPOSITION c [openEHR-EHR-COMPOSITION.vaccination_list.v0]

2. Select Composition Name and Context Start Time with Aliases

This query retrieves high-level data from the composition and uses the AS keyword to assign user-friendly names to the result columns.


SELECT
  c/uid/value,
  c/name/value AS composition_name,
  c/context/start_time/value AS recorded_date
FROM
  EHR e
  CONTAINS COMPOSITION c [openEHR-EHR-COMPOSITION.vaccination_list.v0]


Intermediate Queries
3. Retrieve the Time of Each Immunization Action

This query uses a CONTAINS clause to navigate into the ACTION archetype and select the administration time of the medication.


SELECT
  med_ac/time/value AS administration_time
FROM
  EHR e
  CONTAINS COMPOSITION c
  CONTAINS ACTION med_ac [openEHR-EHR-ACTION.medication.v1]


4. Filter Immunizations Administered Before a Specific Date

This query introduces the WHERE clause to filter the results, finding only immunizations that occurred before the year 2010.


SELECT
  med_ac/time/value
FROM
  EHR e
  CONTAINS COMPOSITION c
  CONTAINS ACTION med_ac [openEHR-EHR-ACTION.medication.v1]
WHERE
  med_ac/time/value < '2010-01-01T00:00:00Z'


5. Select the Specific Immunization Item Code

This query demonstrates how to navigate complex paths that include at-codes to retrieve a specific coded text value from within the ACTION/description.


SELECT
  med_ac/description[at0017]/items[at0020]/value/defining_code/code_string AS immunization_code
FROM
  EHR e
  CONTAINS COMPOSITION c
  CONTAINS ACTION med_ac [openEHR-EHR-ACTION.medication.v1]
  

6. Retrieve Data from the other_context (Admin Cluster)

This query shows how to access data stored in the composition's context, specifically targeting the admin_salut cluster to get the publishing center code.


SELECT
  admin_salut/items[at0007]/items[at0014]/value/defining_code/code_string AS publishing_center
FROM
  EHR e
  CONTAINS COMPOSITION c
  CONTAINS CLUSTER admin_salut [openEHR-EHR-CLUSTER.admin_salut.v0]

Advanced Queries
7. Find EHRs with a Specific Vaccine Code and Performer ID

This query combines two WHERE conditions with AND to find EHRs that received a specific vaccine (code 'MCC') administered by a specific performer (ID '01817273').


SELECT
  e/ehr_id/value
FROM
  EHR e
  CONTAINS COMPOSITION c
  CONTAINS ACTION med_ac [openEHR-EHR-ACTION.medication.v1]
WHERE
  (med_ac/description[at0017]/items[at0020]/value/defining_code/code_string = 'MCC'
  AND med_ac/other_participations/performer/identifiers/id = '01817273')
  AND med_ac/other_participations/performer/identifiers/id = '01817273'

8. Select Nested Medication Details and Order by Time
This query navigates through multiple nested archetypes (ACTION contains a CLUSTER) to retrieve the detailed medication name code and orders the results by the administration time in descending order.

SELECT
  med_ac/time/value AS administration_time,
  medication_details/items[at0132]/value/defining_code/code_string AS medication_code
FROM
  EHR e
  CONTAINS COMPOSITION c
  CONTAINS ACTION med_ac [openEHR-EHR-ACTION.medication.v1]
    CONTAINS CLUSTER medication_details [openEHR-EHR-CLUSTER.medication.v2]
ORDER BY
  med_ac/time/value DESC

9. Find Compositions Based on Context and Content Data Simultaneously

This query retrieves compositions that meet criteria in both the administrative context (admin_salut cluster) and the clinical content (ACTION archetype).


SELECT
  e/ehr_id/value,
  c/context/start_time/value AS composition_date
FROM
  EHR e
  CONTAINS COMPOSITION c [openEHR-EHR-COMPOSITION.vaccination_list.v0]
    CONTAINS (
      CLUSTER admin_salut [openEHR-EHR-CLUSTER.admin_salut.v0] AND 
      ACTION med_ac [openEHR-EHR-ACTION.medication.v1]
    )
WHERE
  admin_salut/items[at0010]/items[at0017]/value/defining_code/code_string = 'E08665478'
  AND med_ac/ism_transition/current_state/defining_code/code_string = '245'

10. Complex Query: Full Correlation with Multiple Selections and Filters

This comprehensive query links information across the entire composition. It finds vaccination compositions from a specific publishing center ('E08665478') where a specific vaccine ('MCC') was administered after 2005, selects data from both the context and the action, and orders the results.


SELECT
  c/uid/value AS composition_uid,
  admin_salut/items[at0001]/value/value AS authorization_date,
  med_ac/time/value AS administration_time,
  med_ac/description[at0017]/items[at0020]/value/value AS vaccine_name,
FROM
  EHR e
  CONTAINS COMPOSITION c [openEHR-EHR-COMPOSITION.vaccination_list.v0]
    CONTAINS (
      CLUSTER admin_salut [openEHR-EHR-CLUSTER.admin_salut.v0] AND 
      SECTION [openEHR-EHR-SECTION.immunisation_list.v0]
        CONTAINS ACTION med_ac [openEHR-EHR-ACTION.medication.v1]
    )
WHERE
  admin_salut/items[at0007]/items[at0014]/value/defining_code/code_string = 'E08665478'
  AND med_ac/time/value > '2005-01-01T00:00:00Z'
  AND med_ac/description[at0017]/items[at0020]/value/defining_code/code_string = 'MCC'
